================================================================================
                              核心问题回顾
================================================================================

问题表现：
  ❌ 系统因脚本块架构混乱而完全崩溃
  ❌ 采用"打补丁"方式导致可运行性被忽视
  ❌ 新功能开发时缺乏系统级集成测试
  ❌ 没有代码审查流程防止脆弱改动进入主分支

根本原因：
  1. 功能开发快速但缺乏质量保证
  2. 没有统一的编码规范
  3. 没有完整的集成测试流程
  4. 没有规范的代码审查流程

代价：
  ⏱️  数小时的调试时间
  💔 整个功能（SEO 设置）前端代码丧失
  📉 用户无法访问后台系统

================================================================================
                              核心规则速记
================================================================================

强制规则（违反即拒绝 PR）：
  ❌ 脚本块 > 2 个
  ❌ HTML 标签不平衡
  ❌ 使用 inline event handlers
  ❌ 硬编码 API URL
  ❌ 直接使用 fetch 而非 authenticatedFetch

推荐规则：
  ✓ 函数命名规范
  ✓ 完善的错误处理
  ✓ 有意义的代码注释
  ✓ 充分的测试覆盖
